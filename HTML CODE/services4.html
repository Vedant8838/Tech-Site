Cybersecurity

Cybersecurity is the practice of protecting devices, networks, and data from digital attacks. It also involves preventing unauthorized access to personal information.

    • • Network Security – Firewall/UTM/NGFW, IDS/IPS, Sandboxing

  • Firewall

    • security device that monitors and controls incoming and outgoing network traffic, acting as a barrier between trusted internal networks and untrusted external networks, allowing only authorized traffic to pass through based on predefined rules.

  • UTM(Unified Threat Management):

    • A single security appliance that combines multiple security features like a firewall, intrusion detection/prevention system (IDS/IPS), antivirus, content filtering, and VPN, providing comprehensive network protection under one device.

  • NGFW (Next-Generation Firewall):

    • An advanced firewall that builds upon traditional firewall functionality by incorporating deeper inspection of network traffic, application control, intrusion prevention, and advanced threat analysis, offering more robust protection against modern cyber threats.

  • IDS (Intrusion Detection System):

    • A system that passively monitors network traffic for suspicious activity, detecting potential security breaches and generating alerts without actively blocking them.

  • IPS (Intrusion Prevention System):

    • A security system that actively monitors network traffic, not only detecting potential threats but also taking action to block them, preventing intrusions from occurring.

  • Sandboxing:

    • A security practice where untrusted code or applications are executed in a isolated, controlled environment to analyze their behavior without risking harm to the main system, allowing for safe evaluation of potentially malicious software.


    Key differences:

  • Firewall (basic): Primarily filters traffic based on pre-defined rules.
  • UTM (combined): Offers a suite of security features within a single device.
  • NGFW (advanced): Includes deep packet inspection, application control, and advanced threat analysis beyond basic firewall functions.
  • IDS (passive): Detects suspicious activity and alerts.
  • IPS (active): Detects and blocks potential threats.



• End Point Security – AV/NGAV/EDR, DLP, Encryption

Endpoint Security" refers to a set of security measures implemented on individual devices like laptops, desktops, and mobile phones to protect sensitive data from unauthorized access, leakage, or loss; key components include:

  • AV (Antivirus):

    • A traditional security tool that detects and removes known malware on a device.

  • NGAV (Next-Generation Antivirus):

    • An advanced antivirus that uses behavioral analysis, machine learning, and other techniques to proactively identify and block even unknown threats.

  • EDR (Endpoint Detection and Response):

    • A system that continuously monitors endpoint activity, detects suspicious behavior, and allows for rapid incident response to potential threats.

  • DLP (Data Loss Prevention):

    • A security feature that monitors data access and transfer on endpoints to prevent sensitive information from being leaked or misused, including through unauthorized copying or sharing.

  • Encryption:

    • A process where data on an endpoint is scrambled using a cryptographic key, making it unreadable to unauthorized users without the correct decryption key, thus protecting sensitive information in case of device loss or theft.


    Key points to remember:

  • Endpoint security focuses on individual devices:

    • Unlike network security which protects the network perimeter, endpoint security directly safeguards data on individual devices.

  • Layered approach:

    • Combining different endpoint security components like AV, NGAV, EDR, and DLP provides a more comprehensive protection strategy.

  • Data protection at rest and in motion:

    • Encryption is particularly important for protecting sensitive data both when it is stored on a device and when it is being transferred.




• Email Security

  • Email security is the practice of protecting your email accounts and data from cyberattacks. It includes using technology to scan incoming emails for threats and encrypt outgoing emails.

      • Types of email security:

    • Cloud email security: A range of services and technologies that protect against email-borne threats.
    • Data loss prevention (DLP): Identifies sensitive content in an email and blocks it from being leaked.

      • How does email security work?

    • Encryption

      • Encrypts emails so that only intended recipients can read them. This helps prevent data breaches and unauthorized access.

    • Spam filtering

      • Filters out unwanted emails like spam and phishing messages.

    • Phishing prevention

      • Uses algorithms and pattern recognition to identify and block emails that attempt to trick users into revealing personal information.

    • Attachment and link scanning

      • Scans attachments and URLs in real-time to identify and prevent malicious files from reaching users.


      Why is email security important?

    • Email security is important because it helps protect your data, users, and organization from cybersecurity attacks.



    • Application Security – Multifactor Authentication, Web Application Firewall(WAF), SSL/TLS

    Application security, or AppSec, is the protection of software applications from attack. It involves using tools and practices to defend against threats like insecure data management, authentication risks, and code injection.


      Multifactor Authentication (MFA):

    • Enhances security by demanding more than just a password to login, often including a code sent to a phone or email, a security question, or a biometric scan.

      • Web Application:

    • A software application that is accessed through a web browser, like an online banking platform or an e-commerce store.

      • Web Application Firewall (WAF):

    • A security system that monitors incoming traffic to a web application, analyzing it for malicious patterns and blocking suspicious requests to prevent attacks like SQL injection or cross-site scripting (XSS).

      • SSL/TLS (Secure Sockets Layer/Transport Layer Security):

    • A cryptographic protocol that encrypts data transmitted between a server and a client, protecting sensitive information like login credentials and credit card details during online transactions.


      • Key points to remember:

    • MFA adds an extra layer of protection: by requiring multiple forms of verification, making it harder for unauthorized users to access an application.
    • A WAF acts as a guard: between a web application and the internet, filtering out malicious traffic before it reaches the application.
    • SSL/TLS is crucial for secure data transmission: over the internet, ensuring privacy and integrity of information.



    • Information or Data Security – Backup

  • In information or data security, a "backup" refers to the process of creating a duplicate copy of important data, typically stored on a separate device or location, to safeguard against data loss in case of a system failure, accidental deletion, cyberattack, or other disruptions, allowing for data recovery if needed; essentially, it's a safety net to ensure critical information remains accessible even if the primary source is compromised.

      • Types of backups:

    • Full backup: Copies all data from the source at once.
    • Differential backup: Copies only data changed since the last full backup.
    • Incremental backup: Copies only data changed since the previous backup, regardless of whether it was a full or differential backup.

      • Key points about backups:

      Purpose:

    • To protect data integrity and availability by providing a readily accessible copy in case of an incident.

      • Process:

    • Involves copying data from its primary location to a secondary storage device, like an external hard drive, cloud storage, or dedicated backup server.



    • Disaster Recovery (DR)/Business Continuity Planning (BCP)

  • Disaster Recovery (DR) refers to a plan outlining how an organization will restore its critical IT systems and data back to operational status after a major disruptive event like a natural disaster or cyberattack, while Business Continuity Planning (BCP) is a broader strategy that focuses on maintaining essential business functions during and following any disruption, including smaller incidents, by utilizing contingency plans and alternative operational methods across all aspects of the business, not just IT systems.

      • Key differences:

      Focus:

    • DR primarily focuses on restoring IT infrastructure and data, while BCP encompasses all business operations, including staff, facilities, and supply chains.

      • Scale of events:

    • DR plans are typically designed for large-scale catastrophic events, whereas BCPs can address smaller disruptions as well.

      • Implementation:

    • DR often involves technical procedures like data backups and system failover, while BCP may include employee relocation, communication protocols, and vendor management strategies.

      • In summary, DR is a subset of BCP, concentrating on the technical recovery of IT systems following a disaster, while BCP takes a holistic approach to ensure the continuity of all critical business functions during and after a disruption.




    • Operational Security

  • Operational security (OPSEC) is a process for protecting sensitive information from unauthorized access. It's a risk management strategy that involves identifying and controlling indicators that could be used by an adversary.

      • How does OPSEC work?

    • Identify critical information: Determine what information is important and could be used against you
    • Analyze threats: Consider what actions could compromise your operations
    • Analyze vulnerabilities: Identify potential weaknesses in your systems and processes
    • Assess risks: Determine which vulnerabilities are unacceptable and could lead to serious consequences
    • Apply countermeasures: Implement measures to reduce or eliminate the risk

      • Why is OPSEC important?

    • OPSEC helps prevent sensitive information from falling into the wrong hands
    • OPSEC can help protect individuals by limiting what they share online and securing their devices
    • OPSEC can help organizations protect their systems and processes from malicious attacks

      • OPSEC was originally used by the military, but it's becoming more popular in the private sector as well.




    • End-User Training

  • End-user training is the process of teaching customers how to use a product or service. It can help users become more confident and competent in using the product, which can lead to increased productivity and fewer support requests.

      • Types of end-user training

    • Face-to-face training
    • Online or e-learning
    • Self-service training
    • In-app, on-the-job adoption training

      • Benefits of end-user training

      Improves accuracy

    • When users understand the key concepts and processes of a product, they can make fewer errors

      • Increases adoption

    • When users are confident and competent in using a product, they are more likely to adopt it

      • Reduces support requests

    • When users are confident and competent in using a product, they are less likely to need support
